El problema que resolvemos

~30%
Retrabajo por sprint
Gap entre diseno y desarrollo

Ciclo tradicional: Figma → handoff estatico → dev interpreta → QA detecta gaps → vuelta a diseno

Sintomas detectados

  • Diseno no valida interaccion real hasta staging
  • Componentes del DS no se usan consistentemente
  • Cada ticket se prototipa sin contexto de navegacion
  • Stakeholders no ven avance hasta deploy
  • Multiples interpretaciones de un mismo Figma

RPP: La solucion

Un pipeline que conecta Jira, Figma y el Design System para generar prototipos funcionales con IA

Valor para los equipos de desarrollo

El prototipo no es un mockup — es un insumo adelantado para implementacion

📂

Codigo base reutilizable

Prototipos usan componentes reales del DS. Dev recibe estructura, layout y bindings que puede adaptar directamente.

✅

Clean Code desde el origen

Control de versiones, naming conventions, arquitectura Angular 18 + Nx desde linea 1.

🕑

Reduccion de ciclo

Dev recibe PR maquetado, validado por diseno, con data mock. Solo queda conectar servicios reales.

📈

Trazabilidad completa

Ticket Jira → Branch → PR → Review → Merge. Nada se pierde en el handoff.
